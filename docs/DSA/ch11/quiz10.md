# Quiz 10
Submit a PDF, Word Document, or Text Entry with the answers to these questions.

## Questions
1. Order the three O(n<sup>2</sup>) sorting algorithms from slowest to fastest in real-world performance (3 points)

2. Multiple Choice: Which sorting algorithm becomes more efficient when data is partially sorted? (1 point)
    A. Insertion
    B. Heap
    C. Selection
    B. Bubble

3. Short Answer: For the algorithm that performs better on partially sorted data, state and prove the best case runtime when the data is already sorted. (3 points)

4. Short Answer: Beyond the heap sort algorithm we'll discuss next week, we can trivially sort elements by inserting them all into a priority queue then popping them off the queue. Based on your knowledge of the priority queue data structure, is this faster (in terms of Big-O) than O(n<sup>2</sup>)? Is there any drawback to this approach compared to the O(n<sup>2</sup>) algorithms? (5 points)

5. Short answer: Assume we have sorted a list of elements. What is the most efficient method we can use to find any element in the list? What is the time complexity of this? (3 points)
